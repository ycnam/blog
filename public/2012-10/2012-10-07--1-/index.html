앞으로 1년간 공부하고 싶은 것

Sunday, Oct 7, 2012

일요일 밤의 우울을 달래기 위해, 그냥 정리해보는 리스트.

  1. Documentation
    많은 사람들이 짧은 시간 안에 쉽게 읽을 수 있고
    쉽게 배포할 수 있으며
    그러면서도 꼭 필요한 사람 외에는 접근을 차단하여 보안 유지가 가능하고
    수정도 용이하고
    문서에 대한 사람들의 피드백도 쉽게 수집할 수 있고
    그래서 결론적으로, '지속 가능한'
    문서를 만드는 기술.
    wiki? git?
    -
  2. web front-end development
    특별한 이유는 없고 그냥 재미가 있음
    여러모로 쓸모가 많을 것 같다
    특히 프로토타이핑할 때 유용하지 않을까
    html + css + js
    - 
  3. Python
    근원을 알 수 없는 '프로그래밍'에 대한 로망
    그 중 왠지 제일 멋있어보임
    특히 '파이썬의 도'가 멋짐
    최근 진도가 잘 안나가고 있음
    -
  4. Design Pattern
    누군가는 이걸 배우고 싶다고 하니
    3년 후쯤 해도 늦지 않다고 했는데...
    그렇게 심오한 것인가?
    일단 '패턴', '재사용' 이런걸 좋아하기 때문에
    (게으른건가? 비효율적인 반복을 싫어한다고 해두자)
    일단 헤드퍼스트를 질러놓음.
    책은 질러놓으면 적어도 3년 안에는 읽게 되니까
    -
  5. 통계
    '직관'으로 파악하지 못하는 '전체'를 보는 눈
    이젠 통계적 지식이 없으면 컴플렉스에 걸릴 지경
    뭐부터 해야할지 모르겠으니 일단 헤드퍼스트 지름
    이건 좀 급해서 읽기 시작.
    -
  6. 포토샵
    포토샵을 공부한다는 것이 왠지 '창피한'일이라고 생각했던
    때가 있었으나, 이제 포토샵 '장인'이 등장하고 있어
    욕심나지 않을 수 없다. 장식예술에 대한 열망
